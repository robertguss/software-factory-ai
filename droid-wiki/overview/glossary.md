# Glossary

Project-specific terms used across Conveyor documentation and code.

| Term                       | Definition                                                                                                                                                                                   |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Acceptance Calibration** | Station that verifies locked acceptance tests fail on the clean base (red) before implementation, proving they are meaningful.                                                               |
| **Agent Brief**            | The packet handed to the implementer agent, containing the slice contract, context pointers, and output schema.                                                                              |
| **AgentSession**           | A recorded session of an AI coding agent (Pi/Codex/Claude/Gemini) executing inside a sandbox.                                                                                                |
| **AGENTS.md**              | Project instruction file generated by Conveyor. Defines conventions, anti-patterns, and coding guidelines for agents working in the repo.                                                    |
| **Autonomy Level**         | A policy dial (L0-L4) controlling what authority an agent run has: planning only, local implementation, PR generation, auto-merge, auto-deploy. See `AUTONOMY_LEVELS.md`.                    |
| **Baseline Health**        | Station that runs clean checkout baseline suites to verify the project starts green.                                                                                                         |
| **Battery**                | Live sampling and measurement studies that provide statistical quality signals for the gate.                                                                                                 |
| **Bead**                   | A unit of work tracked in the `br` issue tracker. The `br` CLI is the source of truth for implementation work in this repo.                                                                  |
| **Blob Store**             | Content-addressed artifact storage. Blobs are keyed by SHA-256 digest.                                                                                                                       |
| **Canary**                 | Known-bad mutant run through the same gate path to verify the gate catches regressions. If a canary passes, the build fails.                                                                 |
| **Cassette**               | Recorded causal transcript of a station run, used for replay and freshness checks.                                                                                                           |
| **CodeScent**              | Code quality analysis adapter that computes health metrics on the diff.                                                                                                                      |
| **Conductor**              | The deterministic BEAM layer that owns state, policy, evidence, and gate verdicts.                                                                                                           |
| **Contract Lock**          | An immutable, machine-checkable acceptance contract that locks the public interface, required tests, and definition of done for a slice. Authored by a different actor than the implementer. |
| **Contract Forge**         | The subsystem that drafts contract locks from plan requirements and agent briefs.                                                                                                            |
| **Context Scout**          | Station that gathers repository context (rg, CodeScent, optional read-only agent pass) for prompt assembly.                                                                                  |
| **Credential Broker**      | Service that issues short-lived, scoped credential leases to agent containers. Raw secrets are never injected directly.                                                                      |
| **Dossier**                | Human-readable run report containing diff, acceptance mapping, commands, quality delta, reviewer verdict, and gate result. Cites machine artifact digests.                                   |
| **Effect Attempt**         | A recorded attempt to produce a side effect (file write, git operation, network call) during a station run.                                                                                  |
| **Effect Receipt**         | A receipt confirming an effect was applied, with digest verification.                                                                                                                        |
| **Emergency Stop**         | Global mechanism to halt all running conveyor operations. Defined in `lib/conveyor/emergency_stop.ex`.                                                                                       |
| **Epic**                   | A grouping of related slices in a plan.                                                                                                                                                      |
| **Evidence**               | Independent, content-addressed proof that a slice met its acceptance criteria. Agent claims are input, not proof.                                                                            |
| **Gate**                   | Deterministic verification that composes stage results into a pass/fail verdict. The gate is the human's stand-in.                                                                           |
| **Gate Canary**            | Mutant gate-only checks that verify the gate catches known-bad changes.                                                                                                                      |
| **Ledger**                 | Append-only audit log of all conveyor events. Idempotent writes keyed by `idempotency_key`.                                                                                                  |
| **Plan**                   | A human-authored decomposition of work into requirements, epics, and slices.                                                                                                                 |
| **Plan Audit**             | Structural and traceability audit of a plan before work graph lowering.                                                                                                                      |
| **Policy Profile**         | A named set of permissions (explore, implement, verify, release, maintenance) controlling what agents can do in a sandbox.                                                                   |
| **Post-Integration Check** | Station that verifies the slice after integration into the target branch.                                                                                                                    |
| **Prompt Builder**         | Station that assembles the implementation prompt with explicit instruction-source trust labels.                                                                                              |
| **Qualification**          | The process of verifying that a plan, contract, or run meets the prerequisites for execution.                                                                                                |
| **Readiness**              | Station that checks whether a slice is ready to execute (dependencies satisfied, contracts locked, policy loaded).                                                                           |
| **Retrospective**          | Station that produces a failure taxonomy and learning summary after a run completes.                                                                                                         |
| **Run Attempt**            | One execution attempt of a slice through the station pipeline.                                                                                                                               |
| **Run Bundle**             | The projected run directory containing manifest, dossier, evidence, review, gate result, provenance, and PR-body draft.                                                                      |
| **RunSpec**                | The immutable, content-addressed input object for one execution attempt. Freezes all inputs that affect evidence validity.                                                                   |
| **Sandbox**                | Isolated Docker container where agent execution happens. Non-root, no Docker socket, no host home mount, network-restricted.                                                                 |
| **Slice**                  | The atomic unit of work, carrying a contract that locks the public interface, required tests, and definition of done.                                                                        |
| **Station**                | A stage in the pipeline (readiness, baseline health, context scout, prompt build, agent session, evidence recording, review, gate).                                                          |
| **Station Run**            | A recorded execution of one station for one run attempt.                                                                                                                                     |
| **Test Pack**              | The locked set of tests that must pass for a slice to be accepted. Read-only during implementation.                                                                                          |
| **Tool Invocation**        | A recorded command or tool call with normalized command spec, policy decision, and output digest.                                                                                            |
| **Toolchain Profile**      | Pinned container image, SBOM, and tool versions for a station.                                                                                                                               |
| **Traceability**           | The ability to trace from requirement to slice to evidence to gate result.                                                                                                                   |
| **Work Graph**             | The dependency-ordered graph of slices derived from a plan.                                                                                                                                  |

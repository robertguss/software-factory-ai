# ADR-19: Mutation/reference-solution and compiler-falsifier policy

Status: Accepted

Date: 2026-06-19

Bead: `software-factory-ai-aamg.1.19`

Gated milestone: P2-B2

## Context

Conventional code mutation scoring assumes a working implementation. At contract
lock time, before an implementer exists, using a hidden reference solution
generated by the same planning system would couple contract authoring to
implementation and create circular evidence of test strength.

Correction D rejects universal mutation testing at contract lock unless there is
a legitimate independent reference implementation. Law 16 forbids circular
test-strength proof, and law 45 requires compiler-derived falsifiers to
establish a non-model floor.

## Decision

Universal code mutation testing is hard-blocking at contract lock only when a
legitimate independent reference implementation exists. Otherwise, conventional
code mutation follows the candidate implementation and is not allowed to
masquerade as pre-implementation proof of contract quality.

Before implementation, Phase 2 hard-gates calibration, hermeticity,
repeatability, base behavior, obligation mapping, compiler-derived falsifiers,
and adversarial loophole review.

For each machine-checkable acceptance criterion, the compiler must create or
validate a concrete falsifying condition. Where structured examples, forbidden
behaviors, properties, metamorphic relations, or interface schemas allow it, a
pure compiler pass emits falsifier seeds independent of the Test Architect. The
Test Architect must preserve those seeds, translate them, or explicitly
supersede them with stronger approved evidence. Dropping a falsifier is an
integrity failure.

## Consequences

The system avoids false confidence from self-authored hidden solutions. Contract
quality rests on independent role separation, oracle feasibility, calibrated
base behavior, hermetic repeatability, obligation mapping, and falsifier
preservation.

Some contracts will not receive early universal mutation scores. That is
acceptable; unsupported checks report their real status instead of defaulting to
pass. Human-only evidence remains human-only and caps autonomy where automation
would be dishonest.

## Implementation Notes

P2-B2 implements the isolated test-only workspace, TestSpecification, TestPack,
challenge artifacts, falsifier translation and preservation, oracle-feasibility
classification, obligation-stage satisfaction, integrity Sentinel integration,
and honest human-verification path.

Acceptance must prove that the Test Architect cannot edit source, tests map to
obligations and ACs with base reasons, dropped falsifiers block,
boundary_unclear routes to split or clarification, universal mutation is
required only with a legitimate reference, human-only evidence remains
human-only, and weak evidence routes to its author rather than the implementer.
